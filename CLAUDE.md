# 松山避難マップ 実装セッション指示書

このフォルダ（`~/Downloads/matsuyama-hinan-map`）で開いた Claude Code セッションは**この実装の owner**。設計の正は `上流からの指示.md` の初期仕様＋これから起こす `docs/spec.md`。

## セッション開始時に必ず確認（自動）
1. **`上流からの指示.md`**（dir直下）＝上流ハブ（cc-research）からの指示・申し送り。未対応があれば対応し、各項目に「対応済み（日付・要点）」を追記する。
2. `docs/spec.md`（実装の正・まだ無ければ初期仕様から起こす）。

## 上流への連絡（上り）
- 悩み・質問・節目の報告は `~/Downloads/cc-research/目安箱/matsuyama-hinan-map.md` に日付つきで追記する（返答は `上流からの指示.md` 経由）。
- **完了・節目の closeout（必須・省略不可）**: 一塊の作業や指示対応が終わったら、①対応した指示に `上流からの指示.md` で「対応済み（日付・要点）」を追記 ②`目安箱/matsuyama-hinan-map.md` に一行報告——の**両方**を必ず行う。実装だけ進めて報告を省略しない（ハブはチャットが見えず目安箱が唯一の合図）。
- **cc-research 内はこの目安箱以外を読まない・書かない**（上流の戦略情報は実装に不要）。

## 役割境界
- 実装の手を動かすのはこのセッション。ハブは方針・リスク・整合性に責任を持つが実装介入（L3）はしない。
- **上流へ上げる前に、自分でできる安い診断は先に済ませる**（再現実行・仕様確認・データ目視・原因の絞り込み）。上流（本人/ハブ）へ上げるのは本当に上流案件だけ＝①仕様の意図・プロダクト判断 ②データ採否やライセンス解釈の判断 ③事実の裏取り（外部権威との照合＝ハブの調査レーン）。

## 環境・規律
- Bash は POSIX 構文（PowerShell構文不可）。Python出力の cp932 エラーは `PYTHONIOENCODING=utf-8 python ...` 前置。
- **質問・確認は本文に箇条書きで出す。AskUserQuestion 等の選択肢UIは使わない**（本人環境で機能しない＝直前本文が隠れる・ツール呼び出しが壊れる既知事象）。

## プロジェクト固有の規律
- **枠＝市民貢献／ポートフォリオ（試験）＋県庁インターン土産・非収益**。課金/広告なし。
- **静的サイト・公開データのクライアント側読込のみ**。葉・可逆系＝モジュール自動進行＋完了ごと自動commitでよい（**push と公開デプロイのみ本人手動**＝外向き）。
- **データのライセンス順守を最優先**（出典表示を実装・各データ個別ページで利用前確認・愛媛県A33の商用/再配信条件は断定せず確認）。
- **「ここは安全／危険」の独自判定は載せない**＝公的指定データの提示に徹する。指定漏れ斜面・0次谷の危険判定はスコープ外。
- **v1スコープ堅持**：IN＝避難場所＋災害種別フィルタ＋ハザード重ね／OUT＝自前の経路探索（外部地図リンクに丸投げ）。
