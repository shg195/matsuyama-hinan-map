> 本ファイルは実装の**正**。上位の文脈・申し送りは `../上流からの指示.md`、セッション運用規律は `../CLAUDE.md` を参照。
> 競合した場合の優先順位：`上流からの指示.md`（スコープ・規律の正） > 本 spec（実装詳細の正） > 実装コード。

## 1. 概要・目的

松山市民が「自宅（や今いる場所）を起点に、**災害種別ごとに『行ける避難場所』と『足元の危険』を1画面で確認**できる」スマホ向け地図アプリ。

- **出自**：cc-research の防災リサーチ2本（松山の構造課題 → 防災選定 → 個人が作れるものの裏取り）で「避難場所＋ハザード表示なら個人が作れる（高確度）」と確認した結果。
- **枠**：市民貢献／ポートフォリオ（試験）＋県庁インターン土産。**非収益・課金/広告なし**。
- **形態**：静的サイト（クライアント側で公開データを読むだけ・サーバー/DB 不要）。

## 2. スコープ（上流指示の確定事項）

### IN（v1）
1. 地図に**現在地（or 住所/地名検索）**と周辺の**指定緊急避難場所**をピン表示。
2. **災害種別フィルタ＝本アプリの肝**。指定緊急避難場所は災害種別ごとに使用可否が異なるため、選んだ災害で「実際に避難先として指定されている場所」だけを表示する。
3. **ハザードレイヤの重ね表示**（土砂災害警戒区域・洪水浸水想定・津波浸水想定）＝自宅が危険区域内か一目で分かる。
4. 避難場所の**詳細**（名称・住所・対応災害種別・想定収容人数 等）。
5. **最寄り避難場所リスト**（直線距離の近い順）＋各場所を**外部地図（Googleマップ等）で開くリンク**。

### OUT（v1では作らない）
- **避難「経路」探索（自前ルーティング）**：徒歩道路ネットワーク／避難所のリアルタイム開設・混雑 API の実在が未確認のため。経路は**外部地図リンクに丸投げ**。経路ナビ化は別途データ裏取り後に判断。
- **危険度の独自判定・予測**：「ここは安全／危険」の独自判定は載せない。公的指定データ（指定緊急避難場所・公的ハザード区域）の**提示**に徹する。指定漏れ斜面・0次谷の危険判定はスコープ外（個人単独 NG・専門家=愛媛大 CDMIR と組む別枠）。

## 3. 規律（最優先・違反不可）

- **データのライセンス順守を最優先**。各データの個別ページで利用前に条件確認。出典表示を必ず実装。
- ⚠️ 愛媛県の土砂災害 A33 の商用/再配信条件は**確定情報なし＝断定せず個別ページで確認**（リサーチで「商用可」と断定する主張は棄却済み）。本件は非商用枠ゆえブロッカーは小さいが、v1 は**ハザードマップポータルのタイル（利用可が明示）を一次採用**し A33 GeoJSON 直読は後送りとする（§5 参照）。
- **独自の安全/危険判定を載せない**（§2 OUT 再掲）。

## 4. 技術スタック（確定）

| 項目 | 採用 | 理由 |
|---|---|---|
| フレームワーク | **Vite + TypeScript**（素・SPA） | 静的・依存最小・GitHub Pages と相性良 |
| 地図ライブラリ | **MapLibre GL JS** | ラスタタイル重ね＋ベクタ表現に強い。先例 mapprint も採用 |
| デプロイ | **GitHub Pages** | 純粋な静的 SPA/PWA・サーバー不要・先例「防災INSIGHT」と同型 |
| データ整形 | ビルド時に **CSV → GeoJSON 変換**（Node スクリプト） | クライアント実行時の整形コストを排除 |

- **UI/ビジュアルデザインは別途 claude design に委譲**。本 spec は**機能要件・画面構成・データ契約**を定義し、配色・レイアウト・コンポーネントの見た目は claude design 側の成果物に従う（§9）。
- **公開（push / デプロイ）操作は本人手動**。実装・ローカル commit はセッション自動進行可。

## 5. データ源（実在確認済み・利用前に各ページで条件再確認）

### 5.1 指定緊急避難場所（ピン本体）
- **松山市オープンデータ CSV `382019_evacuation_space.csv`（CC-BY・自治体標準データセット・危機管理課維持）**
  - メタデータ：https://www.city.matsuyama.ehime.jp/shisei/opendata/metadata/hinanbasho.html
  - 実体 DL：`hinanbasho.files/382019_evacuation_space.csv`（UTF-8 BOM 付き・40列）
  - **実データ確認済み（2026-06-15・本セッション取得）**：データ 390 件／緯度・経度・名称の欠損ゼロ。
- 補完：国土地理院 指定緊急避難場所データ（CSV/GeoJSON）。※網羅性に「最新は市に確認」の限界あり。v1 は松山市 CSV を一次とし、地理院は当面**未使用**（差分照合は将来課題）。

#### CSV 主要列（v1 で使用するもの）
| 列名 | 用途 |
|---|---|
| `ID` | 一意キー |
| `名称` | 表示名 |
| `所在地_連結表記` | 住所表示（例：愛媛県松山市堀之内） |
| `緯度` / `経度` | ピン座標（WGS84） |
| `標高` | 詳細（任意・欠損あり） |
| `災害種別_洪水` | フィルタ（"1"=対応／空欄=非対応） |
| `災害種別_崖崩れ、土石流及び地滑り` | フィルタ（＝土砂） |
| `災害種別_高潮` | フィルタ |
| `災害種別_地震` | フィルタ |
| `災害種別_津波` | フィルタ |
| `災害種別_大規模な火事` | フィルタ |
| `災害種別_内水氾濫` | フィルタ（※下記） |
| `災害種別_火山現象` | フィルタ（※下記） |
| `指定避難所との重複` | 詳細（緊急避難場所と避難所の重複表示） |
| `想定収容人数` | 詳細 |
| `備考` | 詳細 |

#### 災害種別フラグの実分布（2026-06-15 実データ・390件中）
| 種別 | 該当件数 | v1 扱い |
|---|---:|---|
| 地震 | 390 | 表示（全件） |
| 津波 | 331 | 表示 |
| 崖崩れ・土石流・地滑り（土砂） | 319 | 表示 |
| 高潮 | 282 | 表示 |
| 洪水 | 137 | 表示（フィルタの差が最大＝肝の機能を実証） |
| 大規模な火事 | 6 | 表示 |
| **内水氾濫** | **0** | **フィルタ選択肢から除外 or 無効表示**（該当ゼロ） |
| **火山現象** | **0** | **フィルタ選択肢から除外 or 無効表示**（該当ゼロ） |

> 内水氾濫・火山現象は現データで該当 0 件。選択肢に出すと「0件」表示になり混乱を招くため、**v1 はフィルタ UI から除外**を既定とする（将来データ更新で値が入れば自動的に有効化できるよう、件数 0 の種別を動的に非表示にする実装が望ましい）。

### 5.2 ハザードレイヤ（重ね表示）
- **ハザードマップポータルのタイル**（洪水浸水想定・津波浸水想定・土砂災害警戒区域）
  - 利用条件：https://disaportal.gsi.go.jp/hazardmap/copyright/opendata.html （**商用非商用問わず利用可＋出典表示・リンク条件**）
  - v1 はこのラスタ/ベクタタイルを**一次採用**（利用可が明示されているため安全）。
- 土砂災害警戒区域の詳細＝**国土数値情報 A33**（愛媛=コード38）：https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-A33-v1_3_h28.html
  - 利用条件：https://nlftp.mlit.go.jp/ksj/other/agreement.html
  - ⚠️ 商用/再配信条件が未確定のため **v1 では直読を後送り**。ハザードマップポータルのタイルで代替。

### 5.3 下地地図タイル
- **国土地理院 地図タイル**（コンテンツ利用規約への同意が条件）。出典表示を実装。

## 6. 機能仕様（画面・挙動）

> 見た目（配色・余白・コンポーネント意匠）は claude design に委譲。ここでは**何が・どう動くか**を定義する。

### 6.1 画面構成（単一画面・モバイル優先）
- **地図（全面）**：MapLibre GL。初期表示は松山市中心（城山公園付近 33.842, 132.762 を暫定中心）／適切なズーム。
- **位置起点コントロール**：
  - 現在地取得ボタン（Geolocation API）。
  - 住所/地名検索ボックス（ジオコーディング手段は §8 で決定）。
- **災害種別フィルタ**：6種（地震・津波・土砂・高潮・洪水・大規模な火事）の単一選択 or 複数選択トグル。選択に応じてピンを絞り込む。
- **ハザードレイヤ切替**：洪水浸水想定／津波浸水想定／土砂災害警戒区域 の ON/OFF。
- **ピン**：選択中の災害種別に対応する避難場所のみ表示。タップで詳細。
- **詳細パネル**：名称・住所・対応災害種別・想定収容人数・備考・「Googleマップで開く」リンク。
- **最寄りリスト**：起点（現在地 or 検索地点）からの**直線距離**昇順。各項目から詳細・外部地図リンク。
- **出典表示**：常設のクレジット領域（地図タイル・避難場所データ・ハザードの各出典）。

### 6.2 災害種別フィルタの挙動（肝）
- 既定：未選択時は**全件表示**または「災害種別を選んでください」誘導（claude design と擦り合わせ）。
- 種別を選ぶと、その `災害種別_*` 列が "1" のピンのみ表示。
- 複数選択時の論理（AND/OR）は **OR（いずれかに対応）を既定**とする（「この災害なら使える場所」を広く見せるため）。※要レビュー。

### 6.3 距離・外部リンク
- 距離＝**ハバーサイン直線距離**（自前ルーティングはしない＝OUT）。
- 外部地図リンク＝`https://www.google.com/maps/dir/?api=1&destination=<lat>,<lng>`（起点は端末側に委ねる）。

## 8. 決定事項・残課題

### 決定事項（2026-06-15・本人承認）
1. **ジオコーディング手段＝国土地理院 住所検索 API**（`https://msearch.gsi.go.jp/address-search/AddressSearch?q=`）。
   - **無料**（料金・APIキー・登録なし）・**CORS 有効**でブラウザ直呼び可。
   - ⚠️ ただし本来「地理院地図」向けの**半公式**＝継続提供・可用性は保証されず、仕様が予告なく変わりうる。利用は**国土地理院コンテンツ利用規約に従い出典明示（出典：国土地理院）**。
   - 対応：**`geocode()` 等の薄い抽象で隔離**し、将来 Nominatim 等へ差し替え可能にする。
2. **複数選択フィルタの論理＝OR**（いずれかの災害に対応する場所を表示）。
3. **A33 GeoJSON は v1 未採用**（商用/再配信条件が未確定）。ハザードマップポータルのタイルで代替。
4. **`data/raw` の CSV はコミットする**（CC-BY ゆえ再配布可・86KB と小・再現性確保。出典明記必須）。

### 残課題（v1 着手後 or 次段）
- **`想定収容人数` の表記揺れ**（"50,700 " のようにカンマ＋末尾空白）→ CSV→GeoJSON 変換時に正規化。
- **PWA 化の要否**（先例はPWA。v1 はまず SPA、オフライン対応は次段で判断）。
