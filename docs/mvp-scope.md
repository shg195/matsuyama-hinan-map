# v1（MVP）スコープ（docs/mvp-scope.md）

実装範囲の明確化。詳細・正は `spec.md §2`。初版：2026-06-15。

## v1 で作る（IN）
1. 地図に現在地（or 住所/地名検索）と周辺の指定緊急避難場所をピン表示
2. **災害種別フィルタ（肝）**：選んだ災害で実際に指定されている場所だけ表示（OR 論理）
3. ハザードレイヤ重ね（土砂・洪水浸水想定・津波浸水想定／ポータルタイル）
4. 避難場所の詳細（名称・住所・対応災害種別・想定収容人数 等）
5. 最寄り避難場所リスト（直線距離順）＋外部地図（Googleマップ）リンク

## v1 で作らない（OUT）
- 自前の経路探索（ルーティング）＝外部地図リンクに丸投げ
- 危険度の独自判定・予測（公的指定データの提示に徹する）
- 国土数値情報 A33 の直読（商用/再配信条件未確定 → ポータルタイルで代替）
- PWA / オフライン対応（次段で判断）

## 実装順（モジュール）
spec §11 / dev-guide のモジュール一覧に従う：
モジュール0（規約・型）→ 1 scaffold → 2 データ変換 → 3 ピン表示 → 4 フィルタ → 5 ハザード → 6 現在地/検索 → 7 詳細/最寄り → 8 出典 → 9 UI統合 → 全体監査 → デプロイ（本人手動）
