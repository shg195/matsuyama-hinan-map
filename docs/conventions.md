# 共通規約（docs/conventions.md）

全モジュール共通の規約。以降の実装はこれを参照する（参照しない実装は監査で要修正）。初版：2026-06-15。

## 技術スタック前提
- Vite + TypeScript（素・SPA）／ MapLibre GL JS ／ GitHub Pages。

## 命名規約（TypeScript）
- 変数・関数：`camelCase`
- 型・interface・class：`PascalCase`
- 定数（モジュールレベルの不変値・enum値）：`SCREAMING_SNAKE_CASE`、ただし型安全な union/const オブジェクトを優先
- 真偽値：`is`/`has`/`can` 接頭辞（例：`hasHazardLayer`）

## ファイル名・ディレクトリ名
- ファイル名：小文字 `kebab-case`（例：`hazard-layer.ts`）。単一語は小文字（例：`map.ts`）
- ディレクトリ名：小文字（例：`src/ui/`）
- 1ファイル1責務。spec §7 のディレクトリ構成案に従う

## import / export
- **名前付きエクスポートを優先**（`export const` / `export interface`）。`export default` は避ける
- 型のみの import は `import type { ... }` を使う
- 相対 import はモジュール間の依存が一方向になるようにする（循環依存を作らない）

## コメント・ドキュメンテーション
- 日本語で可。**「なぜ」を書く**。自明な「何を」は書かない
- 出典・ライセンス・データ仕様に関わる箇所は根拠リンクをコメントに残す（規律：ライセンス順守）
- 仕様にない TODO は埋め込まない（CLAUDE.md 禁止事項）

## エラーハンドリング基本方針
- 外部依存の失敗（データ取得失敗・Geolocation 拒否/失敗・ジオコーディング失敗・タイル読込失敗）は**握りつぶさず、ユーザー向けに状態を提示**する
- ネットワーク呼び出しは境界（呼び出し元）で catch し、UI にフォールバック表示
- ジオコーディングは `geocode()` の薄い抽象で隔離し、実装差し替え可能にする（spec §8）

## データ・ライセンス規律（最優先）
- 出典表示を必ず実装（地図タイル＝国土地理院／避難場所＝松山市 CC-BY／ハザード＝ハザードマップポータル）
- 「ここは安全／危険」の独自判定は載せない（公的指定データの提示に徹する）

## 座標の扱い
- 内部表現は WGS84。GeoJSON の座標順は **[経度, 緯度]（[lng, lat]）**、アプリ内の個別フィールドは `lat` / `lng` を明示名で持つ
- 距離はハバーサイン直線距離（自前ルーティングはしない＝spec OUT）
