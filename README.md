# 松山避難マップ

松山市民が「自宅（や今いる場所）を起点に、**災害種別ごとに『行ける避難場所』と『足元の危険』を1画面で確認**できる」スマホ向けの地図アプリです。公開データのみをクライアント側で読み込む**静的サイト**で、サーバー／DB は不要です。

- **公開URL（予定）**: https://shg195.github.io/matsuyama-hinan-map/
- **枠**: 市民貢献／ポートフォリオ。**非収益・課金/広告なし**。

> **免責**: 本アプリは公的に指定されたデータ（指定緊急避難場所・公的ハザード区域）を**そのまま提示**するもので、「ここは安全／危険」といった**独自の判定は一切行いません**。実際の避難判断は、松山市・気象庁等の公式情報と現地の状況に従ってください。

## 機能（v1）

- **避難場所のピン表示**: 松山市の指定緊急避難場所を地図上に表示。
- **災害種別フィルタ（本アプリの肝）**: 地震・津波・土砂（崖崩れ・土石流・地滑り）・高潮・洪水・大規模な火事 の6種で絞り込み。指定緊急避難場所は災害種別ごとに使用可否が異なるため、選んだ災害で「実際に避難先として指定されている場所」だけを表示します（複数選択は OR ＝いずれかに対応）。
- **ハザードレイヤの重ね表示**: 洪水浸水想定・津波浸水想定・土砂災害警戒区域の ON/OFF。
- **詳細パネル**: 名称・住所・対応災害種別・想定収容人数・備考・「Googleマップで開く」リンク。
- **現在地／住所検索**: 現在地取得、または住所・地名で起点を指定。
- **最寄り避難場所リスト**: 起点からの**直線距離**（ハバーサイン）の近い順。各項目から外部地図（Googleマップ）リンク。

### v1 で作らないもの（スコープ外）

- **避難「経路」探索（自前ルーティング）**: 経路案内は外部地図リンクに委ねます。
- **危険度の独自判定・予測**: 上記「免責」のとおり、提示に徹します。

## データ出典・ライセンス（順守を最優先）

| データ | 出典 | ライセンス／利用条件 |
|---|---|---|
| 指定緊急避難場所 | [松山市オープンデータ `382019_evacuation_space.csv`](https://www.city.matsuyama.ehime.jp/shisei/opendata/metadata/hinanbasho.html)（危機管理課維持） | CC BY |
| 下地地図タイル・住所検索 | [国土地理院（地理院タイル／地名検索API）](https://maps.gsi.go.jp/development/ichiran.html) | 国土地理院コンテンツ利用規約 |
| ハザードレイヤ（洪水・津波浸水想定／土砂災害警戒区域） | [ハザードマップポータルサイトのタイル](https://disaportal.gsi.go.jp/hazardmap/copyright/opendata.html) | 商用非商用問わず利用可（出典表示・リンク条件あり） |

- 各出典はアプリ画面内に**常設のクレジット表示**として実装しています。
- 利用前に各配布元ページで条件を再確認しています。土砂災害の詳細データ（国土数値情報 A33）は商用/再配信条件が未確定のため、v1 では直読せずハザードマップポータルのタイルで代替しています。

## 技術スタック

| 項目 | 採用 |
|---|---|
| フレームワーク | Vite + TypeScript（素の SPA） |
| 地図ライブラリ | MapLibre GL JS |
| データ整形 | ビルド時に CSV → GeoJSON 変換（Node スクリプト） |
| デプロイ | GitHub Pages（GitHub Actions による自動デプロイ） |

## ローカル開発

Node.js 22 系を推奨します。

```sh
npm install      # 依存インストール
npm run dev      # 開発サーバー（http://localhost:5173/matsuyama-hinan-map/）
```

その他のスクリプト:

```sh
npm run build      # build:data（CSV→GeoJSON）+ 型チェック + 本番ビルド（dist/ 出力）
npm run build:data # 避難場所 CSV から GeoJSON を再生成
npm run typecheck  # 型チェックのみ
npm run preview    # ビルド結果のローカル確認
```

## デプロイ

`master` への push をトリガーに、GitHub Actions（`.github/workflows/deploy.yml`）が自動でビルドし GitHub Pages へ公開します。初回のみ、リポジトリの **Settings → Pages → Source = 「GitHub Actions」** を設定してください。

## ディレクトリ構成

```
data/raw/        変換元の公開データ CSV（再現性のため意図的にコミット）
public/data/     ビルドで生成される GeoJSON
scripts/         CSV → GeoJSON 変換スクリプト
src/             アプリ本体（地図・フィルタ・ハザード・詳細・検索 等）
docs/            spec.md（実装の正）・規約・監査レポート
```
